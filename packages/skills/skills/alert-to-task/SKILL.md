# Alert To Task

## 适用范围

这个子技能用于“先判断是否异常，再决定是否升级为任务”的场景。适合：

- 库存、销量、回款等阈值监控
- 指标异常时自动建任务
- 命中条件后通知负责人

## 必要流程

1. 先定位唯一的数据集。
2. 再读取真实数据并确认字段结构。
3. 再做阈值判断或异常判断。
4. 只有命中条件时，才升级成任务或提醒。

## 必要 CLI 使用

- `search-my-datasets`
- `query-dataset`
- `preview-dataset-data`

## 规则

- 没有真实数据时，不要做阈值判断。
- 不要伪造当前值、阈值、跌幅或负责人。
- 如果条件没有命中，要明确返回“不触发”。
- 如果用户没有指定，不要擅自假设通知渠道。

## 建议输出

- 监控对象
- 当前值
- 判断规则
- 是否触发
- 如果触发，给出建议动作或任务内容
